## 0. Objective

Prove one assembled causal path across the four layers without claiming that
the new router, pressure law, source adapter, coding product or lineage runner is
finished.

```text
L1 continuing domain
-> bounded newborn ingress
-> L2 transient recognition and one exact disposition
-> existing CALM/lower-body mechanics
-> honest MANIFEST and terminal corpse
```

The gate is body-physics evidence. It is not a coding benchmark.

## 1. Isolation Protocol

The integration is selected only by an explicit versioned option:

```lua
options.packet_life = {
  protocol_version = "vertical_packet_life.v0",
  flow_domain = domain,
  projection_adapter = registered_adapter_or_nil,
  relation_policy = registered_policy,
}
```

Absent or unknown protocol:

```text
current packet.new path remains active
current FLOW behavior remains active
current default router_mode=shadow remains active
no L1 domain is touched
no new L2 reader is invoked
```

The opt-in path may initially be used only by deterministic fixtures. It cannot
become the default in the same implementation step.

## 2. Route-Evidence Separation

Roadmap step 4 proves seam execution before pressure repair. Therefore the gate
has two evidence tiers:

```text
V-PHYSICS
  test harness selects an intended canonical adjacent edge
  Packet body validates and commits every edge
  every organ executes through registry/current-tick rights
  authority is marked harness_override
  result counts as seam/organ evidence only

V-ROUTING
  tree pressure selects the edge without fixture authority
  reserved for roadmap step 5
  only this tier may count toward router promotion
```

Direct organ calls without committed position/tick do not pass V-PHYSICS.
Harness-selected routes never enter edge promotion statistics as tree evidence.

## 3. Deterministic Fixture Source And Projection

The gate uses a small explicit Lua 5.4 integer source and a registered fixture
adapter. It does not use the production prompt as L1 seed.

Fixture projection contract:

```lua
{
  protocol_version = "l1.fixture_projection.v0",
  adapter_id = "vertical_pair.v0" | "vertical_single.v0",
  flow_ref = exact_birth_ref,
  units = bounded ordered physical_measurement_units,
  relation_candidates = bounded structural_candidates,
  event_truth_status = "runtime_confirmed",
  content_truth_status = "non_semantic_measurement",
}
```

`vertical_pair.v0` emits two bounded L1 samples with exact stream/birth/index
provenance and one materially declared adjacency/shared-moment candidate.
CONNECT still performs detection and writes the raw epoch; the adapter does not
inject a CALM form, disposition or route.

`vertical_single.v0` emits one sample and therefore grows the honest empty probe
control.

The prompt remains a separate semantic FLOW unit. The flow mark remains audit
provenance and is not itself addressable relation pressure.

## 4. Ingress Sequence

Every opt-in life follows:

```text
validate request/carrier/graves/projection policy
packet_birth.create(flow_domain, prompt, options)
grave.attach(newborn, selected_graves) when present
registry FLOW materializes prompt/carrier/projection/repair inputs
commit first canonical edge
```

For future descendant tests, NETWORK only prepares the request. It is never an
operator tick. User and NETWORK births share the same L1 atomic birth law and
FLOW boundary.
