# procesis-body docs

These docs use the same four-layer discipline as procesis, but for the body.

```text
00_chaos     raw reasoning, unresolved direction, living discussion
01_table     yellowprints: maps, inventories, candidate layouts
02_crystall  blueprints: stable architecture contracts
03_manifest  documentation of what actually exists
```

## Rule

Do not promote a thought directly from chaos to implementation.

```text
chaos -> table -> crystall -> manifest
```

The body is allowed to evolve, but each layer has a different job:

- chaos preserves why a direction appeared;
- table makes it navigable;
- crystall turns it into constraints;
- manifest describes the current executable surface.

## First Chaos Packet

The first packet of body documents is in:

```text
docs/00_chaos/
```

Start there before treating any body idea as a stable spec.
