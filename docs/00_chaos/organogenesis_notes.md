# Organogenesis Notes

The body should not start with a large fixed organ set.

It should be able to grow bounded organs from process pressure.

## Core Thought

Organs are not only predefined modules.

An organ can appear when the current packet cannot continue cleanly with the
existing body.

Sources of organ pressure:

```text
task shape
human operator style
current ProcessLang operator
substrate behavior
unsupported forms
tool/runtime constraints
repeated residue
```

The human does not only provide "what" to do.

The human also provides "how" pressure:

```text
wording
urgency
ambiguity
precision
roughness
trust level
domain
expected artifact shape
```

That pressure affects what organ is needed.

## Substrate-Specific Organs

Different substrates fail differently.

Examples:

```text
DeepSeek may over-generate missing APIs
another substrate may under-plan
another may over-explain
another may need stricter glyph payload
another may work better in natural language
```

So organ birth can depend on:

```text
substrate
operator
task pressure
human style
```

This means the body should not pretend all models need identical organs.

The body identity stays stable.
The organ set can adapt.

## Phantom First

New organs should appear first as phantoms.

```text
phantom organ = bounded temporary organ created for a packet
```

It must have:

```text
parent packet
operator role
birth reason
budget
trace
death
residue
```

It is not an independent agent.

## Crystallization

A phantom organ can become a real organ only if it repeats and proves itself.

Rough path:

```text
unsupported form
-> gap residue
-> phantom organ
-> repeated usefulness
-> table map
-> crystall blueprint
-> tests
-> real organ module
```

No tests, no crystallized organ.

## Why This Matters

Most agents have fixed tools and a prompt.

This body can grow the missing organ from task pressure.

That is different from plugin installation.

Plugin:

```text
external tool added by design
```

Organ:

```text
body part grown from repeated process need
```
