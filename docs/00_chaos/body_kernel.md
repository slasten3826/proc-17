# Body Kernel Notes

`procesis-body` is not a chatbot wrapper.

The intended shape:

```text
procesis      = soul / law / orientation
procesis-body = body / runtime / organs
LLM substrate = current / spark / active fire
task          = mortal packet life
tools/repo    = environment / limbs
```

The body must keep its identity across substrate changes.

If DeepSeek, GLM, Claude, Codex, local llama, or another model can be swapped and
the agent changes only in quality/noise/latency, then the body is real.

If changing model changes the agent's basic law, then the model was the body and
the architecture failed.

## First Body Role

The first real body should be a coding agent.

Minimum living loop:

```text
receive task
birth packet
read repo/context
route through organs
ask substrate only at explicit boundaries
validate claims against runtime truth
edit files through tools
run commands/tests
manifest result
write residue
end or die
```

This is intentionally close to Codex/Claude Code/opencode style agents, but the
identity should sit in the body, not in the substrate.

## Two Centers

Old Eva placed too much in one center.

New body has two centers:

```text
☴ OBSERVE = chaos-facing reader
☱ RUNTIME = manifest-facing sustainer
```

OBSERVE reads what appears.

RUNTIME tracks what can continue.

They are related, but not the same organ.

## Non-Negotiable Distinction

```text
semantic output != runtime truth
```

The substrate can emit plausible text.
The body must check whether the world can sustain it.

This is where tools, files, tests, permissions, cost, and trace matter.

Without this boundary, the agent is just language pretending to be execution.
