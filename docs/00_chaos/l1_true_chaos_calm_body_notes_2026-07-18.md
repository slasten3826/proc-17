## 0. One Sentence

proc-17 built most of the machine that distinguishes, holds, validates, cycles,
and manifests forms, but omitted the live CHAOS field beneath that machine.

That field is named simply:

```text
L1
```

## 1. The Name Collision

The word `CHAOS` currently names two different things.

### 1.1 True CHAOS

From packet-slop L1:

```text
pre-addressable material
seed-grown medium
self-changing on every tick
position-dependent interaction
irreversible local transformation
no stable form inside the medium
only bounded fingerprints may cross upward
```

This is the canonical meaning of `L1`.

### 1.2 Current `packet.chaos`

Current proc-17 stores:

```text
raw_prompt
fragments
unresolved_pressure
observations
empty fingerprints/drift placeholders
```

These are already named, addressable semantic records. They are not the live
L1 medium. `packet.chaos` is better understood as a semantic ingress/archive
inside an already structured body.

Its code name remains for compatibility. Its architectural meaning is now
explicitly narrower:

```text
packet.chaos != L1
```

## 2. Is Current proc-17 CALM?

At the outer scale, yes.

Current proc-17 contains more than the literal `packet.calm` table. It contains
TABLE, CRYSTALL, runtime, boundaries, mortality, lineage, and MANIFEST. But all
of these regions operate on forms that are already distinguishable,
addressable, versioned, and auditable.

Relative to L1:

```text
L1       = living pre-form material
proc-17  = the CALM-side machine that distinguishes and works on forms
```

This is recursive context, not a request to rename the entire repository or
merge every current region into `packet.calm`.

## 4. One Glyph, More Than One Layer

The packet-slop blueprint repeats glyphs between layers and explicitly says
their meaning changes.

```text
L1-FLOW       movement of material through the medium
L1-CONNECT    continuous positional mixing inside the medium
L1-DISSOLVE   unavoidable replacement/decay of the prior local state

operator ▽    discrete Packet birth/ingress act
L2-CONNECT ☰  bounded recognition of relations between distinguished objects
L2-DISSOLVE ☷ paid release of a distinguished rigid relation or form
```

L1 aspects are not extra operator calls and do not create extra Tree edges.
They are body physics beneath the ten-operator registry.

The existing relation-consumer acceptance law remains correct for
`L2-CONNECT`. It never governed continuous L1 connectivity.

## 5. What This Explains

proc-17 asked one discrete `☰` organ to carry two incompatible expectations:

```text
continuous expectation:
  relation is always happening because the medium is alive

discrete expectation:
  relation is a bounded act requiring exact objects, a reader, and discharge
```

That superposition helps explain why relation pressure oscillated between:

```text
zero     no demonstrated L2 consumer
infinite every new unit appears to demand another CONNECT tick
```

The L2 consumer defect remains real. The newly recovered L1 does not excuse a
writer without a reader. It removes the unrelated continuous burden from the
L2 organ.

## 6. Proposed Body Boundary

Working regions:

| Region | Role | Mutation law |
|---|---|---|
| `L1` | Live pre-addressable medium | Mutates at most once per committed body tick; effect-failure policy remains open |
| `packet.chaos` | Semantic ingress/archive | Append-only records; existing entries do not mutate |
| `packet.field` | Addressable TABLE/potential projection | Operator-owned versioned mutation |
| `packet.calm` | Formed structures/work | Existing CALM contracts |
| `packet.trace` | Body audit history | Append-only and immutable by reference |

The old L1 implementation also calls one mutable array `trace`. proc-17 must
not confuse it with `packet.trace`. The mutable L1-local array needs an
explicit name such as:

```text
l1_trace
```

or another non-audit term selected at crystallization.

## 7. L1 Is Not Yet Pressure

A varying fingerprint is not automatically useful tension.

```text
different prompt -> different fingerprint
```

can be reproduced by an ordinary hash. Therefore the first L1 work proves only
faithful physics and non-collapse. It does not grant routing authority.

Required sequence:

```text
old stand parity
-> deterministic matched controls
-> non-trivial temporal metrics
-> named upper reader
-> shadow causal ablation
-> only then a pressure proposal
```


```text

```

This avoids making one symbol mean both a Packet layer and a test tier.
