## Trigger

`slop.raw.txt` was reread as packet material, not as a general ontology.

This changes the reading.

The file is not only "about ProcessLang".

It is a raw packet-body specification written as ten Python-like modules.

Each module describes one way the packet exists, mutates, loses richness,
continues, validates, stabilizes, and finally dies into output.

## Core Reframe

The packet is not a data bag.

The packet is the process body.

Current proc-17 already has a packet skeleton:

```text
substrate
chaos
boundary
calm
tension
trace
residue
death
manifest
```

But `slop.raw.txt` says what this body must do internally:

```text
▽ birth as pure flow
☰ connection as interweaving
☷ dissolution as removal of rigid form
☴ observation as partial self-sight
☵ encoding as structure-with-loss
☳ choice as collapse-with-dead-alternatives
☲ cycle as continuation drive
☶ logic as executable spell/rule
☱ runtime as stabilized automatic foundation
△ manifest as form-death/output
```

## Operator Reading

### ▽ Flow

The packet begins as dirty flow, not as a plan.

Important fields:

```text
is_flowing
resistance
emergence_potential
ticks
engagement
delta
```

Proc-17 implication:

```text
user prompt -> packet.chaos.flow
```

The prompt should not become a task list immediately.

It should first become flow with pressure.

### ☰ Connect

Connection is not "join strings".

It is interweaving between processes.

Important fields:

```text
recognition_depth
interpenetration_level
reciprocity
boundary_fluidity
shared_presence
connection_intensity
```

Proc-17 implication:

```text
connect should bind packet parts, repo facts, substrate proposals, and task pressure
without pretending they are one object
```

Connection should preserve relation quality.

### ☷ Dissolve

Dissolve detects rigid forms and returns them to flow.

Important fields:

```text
rigidity_level
content_hash
dissolution_potential
fluid_truth
clarity
adaptability
process_alignment
```

Proc-17 implication:

```text
dissolve is not deletion
dissolve is anti-dogma pressure over packet structures
```

It should remove unsupported or too-rigid forms before they become false calm.

### ☴ Observe

Observe splits the process into observer and observed.

It never sees the whole packet.

Important fields:

```text
observer
observed
completeness < 1.0
missing = observer_itself
distance
beauty/intensity
```

Proc-17 implication:

```text
observe may call substrate
observe may inspect packet
observe must keep a missing-piece marker
```

The eye is useful because it is incomplete.

### ☵ Encode

Encode is the first major missing piece in proc-17.

It is not line splitting.

It creates a transferable form and records loss.

Important forms:

```text
HIERARCHY
SEQUENCE
CATEGORY
TEACHING
LANGUAGE
```

Important fields:

```text
encoding_type
loss_percentage
creates_hierarchy
creates_sequence
reversible
hierarchy_lens_visible
original_richness
transmitted_richness
receiver_distorts
```

Proc-17 implication:

```text
semantic chaos -> packet.calm.structure
```

`☵` must produce packet-readable structure, not just alternatives for `☳`.

### ☳ Choose

Choose collapses possibility into actuality.

It kills alternatives.

Important fields:

```text
possibilities
potential
chosen
lost
collapse_type
before
after
reversible = false
```

Proc-17 implication:

```text
☳ should remain dumb
☵ should provide the possibility space
☳ collapses it and records what died
```

If the field is bad, `☳` will look bad.

That is usually an encode failure, not a choose failure.

### ☲ Cycle

Cycle is continuation drive.

It does not decide.

It says "more" while the body can still pay.

Important fields:

```text
intensity
cycles_completed
burnout_risk
is_eternal
total_cycles
```

Proc-17 implication:

```text
cycle should be cheap
cycle checks remaining work and sends the packet back through runtime pressure
```

Cycle should not become a planner.

### ☶ Logic

Logic is spell/rule.

It turns desire into executable constraint.

Important fields:

```text
spell
intention_hash
code
executed
reality_changed
lie_level
invocation_result
```

Proc-17 implication:

```text
logic validates and executes rules
logic does not create truth
logic rejects unsupported form
```

In code work, logic is where sandbox, tests, schema, and repo rules belong.

### ☱ Runtime

Runtime is stabilized automatic foundation.

It makes spells/patterns run without being re-decided every time.

Important fields:

```text
pattern
repetition_count
strength
stability
foundation_state
runtime_eternal
```

Proc-17 implication:

```text
runtime should see the manifest-side state of the packet
runtime should count ticks/cycles/budget/residue
runtime should not be human memory
```

Runtime is the lower eye.

### △ Manifest

Manifest is death into form.

It is not "the assistant says done".

Important fields:

```text
original_process_richness
remaining_richness
is_dead
perceived_as_real
touch_of_keter
```

Proc-17 implication:

```text
manifest outputs only the final form/residue/code/report
manifest must know what richness was lost
```

Output is the corpse or trace of the packet, not the whole packet.

## Main Consequence

The next implementation pressure is packet structure.

Not UI.

Not more prompts.

Not more fixed routes.

Proc-17 needs `☵` to encode chaos into structures described by `slop.raw.txt`.

Then `☳`, `☶`, `☲`, `☱`, and `△` can act on packet structure instead of prose.
