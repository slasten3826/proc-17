## Trigger

The first qualified-pressure treatment gave exact witnesses and body-owned
actions to relation recognition, relation formation, and upper observation.
It deliberately stopped before generic ENCODE and CHOOSE.

The next pressure is not merely:

```text
add two more target glyphs to qualified_pressure.lua
```

The real question is:

```text
what exact fact makes form necessary?
what exact fact makes collapse necessary?
what exact bounded action can discharge each need?
```

Without those answers, the new router would only rename the old coarse rules:

```text
unmapped unit -> ☵
two CALM items -> ☳
```

That would reproduce binary pressure inside a richer envelope.

## 1. Current Runtime Inventory

### 1.1 ENCODE currently has two materially different paths

#### Relation-guided path

`organs/encode.lua` already accepts an exact `relation_input` containing:

```text
raw relation epoch
relation ids
endpoint ids and versions
source event refs
optional requested shape
```

It then:

```text
checks the current raw relation phase
forms CALM work units
creates canonical field units
records a non-shadow identity map
records a relation_formation event
records visible loss
returns exact effect_scope_refs
```

This path is already selected by `relation_formation_need` and an immutable
`pressure.action_plan.v0`. It is the first production-shaped ENCODE consumer
and must remain valid.

#### Generic semantic-text path

The compatibility path currently:

```text
collects packet.chaos fragments or raw_prompt
concatenates them into text
passes the text to logic.encode
splits semantic text into lines/sections
chooses hierarchy/sequence/category/teaching/language through heuristics
writes CALM, work units, field units, loss, and a coarse identity map
```

Its canonical migration is explicitly:

```text
field_shadow.status = shadow_only
identity_map.mapping_kind = coarse_all_sources
identity_map.shadow_only = true
formation_basis = semantic_text
```

This path is useful compatibility archaeology. It is not yet a qualified
body action because:

```text
the source operand is concatenated CHAOS text, not a bounded field view
the identity map names old ids but not the exact source versions
the requested receiver form is inferred from prose vocabulary
the harness may still supply limits and semantic ranking outside route evidence
the output may be human-language structure rather than machine receiver form
```

### 1.2 CHOOSE already has a real effect but a coarse cause

`organs/choose.lua` and `runtime/field.lua` already implement important
physical behavior:

```text
one choice event is append-only
selected field units become activation=selected
killed alternatives become activation=suppressed
each activation mutation increments the exact unit version
CALM work units are not rewritten
choice loss is paid by the Packet
```

The current cause and scope are weaker:

```text
registry readiness reads CALM item ids without versions
all current CALM items may be treated as one possibility set
logic.choose selects by caller semantic_ranking or field order
the activation plan is built inside the organ after routing
the compatibility mapping from legacy item id to field id is shadow-only
the effect payload has no route-owned exact operand scope
```

The current tree router correctly excludes one-item confirmation from a real
CHOOSE candidate, but the organ can still record confirmation when called
directly. That behavior is useful as compatibility output; it must not become
a qualified `choice_need` or mandatory identity loss.

### 1.3 Current loss is visible but not final physics

ENCODE loss currently includes:

```text
encoding type
input/output/omitted counts
truncation and addressable loss_log
provisional loss percentage
```

CHOOSE loss currently maps:

```text
not_chosen_count / before_count
```

The visibility law is already useful and must stay. The numeric laws are still
provisional. Full tree physics says CHOOSE loss ultimately depends on
suppressed potential mass and separation, not only list count.

This treatment must not hide that gap by relabeling the count proxy as measured
identity physics.

## 2. What The Older Pair Contract Still Gets Right

The following laws survive unchanged:

```text
☵ creates addressable form
☳ collapses an explicit possibility field
☵ and ☳ must be designed as a pair
bad CHOOSE behavior usually points back to a bad field
ENCODE records visible loss and identity remapping
CHOOSE records selected and killed alternatives
one remaining alternative is confirmation, not choice
CHOOSE does not validate truth, route, continue, or manifest
ENCODE does not validate the truth of encoded content
```

The following old assumptions require amendment:

```text
direct unbounded CHAOS text is not the production ENCODE operand
language/teaching fallback is not production evidence for a coding body
an uncovered source object alone is not encoding pressure
two unrelated field units are not automatically a possibility field
caller-provided semantic ranking cannot own a qualified CHOOSE action
an organ visit is not discharge without exact effect evidence
count-based choice loss is not final calibrated physics
```

## 3. Code-Only Boundary

The earlier decision that ENCODE is for code must be read precisely.

It does not mean that source carriers can never contain text. User prompts,
model proposals, diagnostics, file contents, and plans may all arrive through
textual carriers.

It means:

```text
qualified ☵ does not improve human prose as an end in itself
qualified ☵ forms a declared Packet-native structure for a machine reader
```

The production output is required to be receiver-suitable:

```text
addressable work
code/repository structure
relations
constraints
alternatives
artifact or evidence references
another explicitly declared Packet-native form
```

`teaching` and `language` remain legitimate historical encodings and may stay
available for compatibility experiments. They do not qualify routing or
promotion until a named machine receiver actually requires them.

The field is not required to be human-readable. It is required to be:

```text
addressable
versioned
provenanced
bounded
usable by a named reader
honest about loss
```

## 4. Encoding Need Hypothesis

### 4.1 Material alone is not need

The existence of a live unencoded field unit is a fact, not automatically an
instruction to encode it.

Qualified encoding pressure requires a causal chain:

```text
exact source object/version exists
AND a named receiver contract accepts that source class
AND the receiver requires a declared form over that bounded scope
AND no current non-shadow formation satisfies that contract at those versions
```

This mirrors the relation treatment:

```text
coverage gap alone != relation need
unmapped material alone != encoding need
```

### 4.2 Observation precedes formation when semantic sight is required

If the receiver contract requires semantic sight, an unobserved source first
creates `upper_observation_need`. Only the current observed version may create
`encoding_need`.

Expected local sequence:

```text
source arrives
-> exact upper observation
-> exact structure formation
-> field-native consequence observation
```

This is not a mandatory hardcoded rail. It follows from two separate needs:

```text
blocking observation demand outranks formation affordance
formation creates a new material version that requires consequence sight
```

### 4.3 Proposed exact encoding need

Working shape:

```lua
{
  kind = "encoding_need",
  target_operator = "☵",
  causal_class = "causal_affordance",
  consumer_contract = string,
  requested_shape = string,
  source_unit_ids = string[],
  source_versions = { [id] = integer },
  scope_refs = exact_source_refs,
  provenance_refs = observation_and_consumer_refs,
  existing_formation_ref = nil,
}
```

`causal_affordance` is the conservative v0 class. A later named invariant may
promote a specific receiver to `blocking_demand`, but this treatment must not
declare all encoding urgent.

### 4.4 Proposed bounded ENCODE action

The action should carry references, not copied arbitrary source text:

```text
exact source ids and versions
named receiver contract
declared requested shape
bounded size/loss policy owned by the body
observation evidence required by that contract
```

The organ re-reads the exact carriers from the field after readiness verifies
the versions. The caller and substrate cannot replace the source scope or
requested shape after the route is committed.

The existing `relation_formation` action remains a specialized ENCODE mode.
The new generic mode should have a distinct name such as:

```text
structure_formation
```

### 4.5 Encoding discharge

An ENCODE visit is not discharge. Discharge requires one effect bundle over
the exact source scope:

```text
accepted crystallization
new canonical field ids
old id + old version -> new ids mapping
receiver contract and requested shape
complete visible loss record
non-shadow identity map
effect refs matching the committed operand refs
```

The resulting formation record becomes the named proof that the exact source
versions have a receiver-suitable form.

### 4.6 Encoding re-arm

The same need reappears only when its own referent changes:

```text
a covered source unit changes version
the requested receiver contract changes
the requested shape changes
the prior formation is shadow-only, malformed, or its output is gone
new source membership is added to the bounded formation scope
```

It must not re-arm because an unrelated unit changed the global potential
revision.

## 5. Choice Need Hypothesis

### 5.1 Multiplicity alone is not choice pressure

Two arbitrary live units do not form a lawful possibility field.

Qualified CHOOSE requires:

```text
one explicit possibility-set provenance
at least two current live alternatives from that same set
a named consumer requiring cardinality collapse
a body-addressable selection basis or policy
```

Examples that do not create qualified choice pressure:

```text
a prompt unit plus a diagnostic unit
two unrelated ENCODE outputs
one live alternative plus one suppressed alternative
one remaining alternative
several items that a receiver permits to coexist
```

### 5.2 The possibility set should come from formation

☵ owns possibility formation. Therefore the cheapest honest v0 set identity
is an immutable formation/identity-map event carrying a declared choice
contract, not a second mutable list.

Working contract:

```text
formation event identifies output unit ids
formation metadata declares which outputs are alternatives
choice contract names the downstream consumer and allowed cardinality
current unit versions and activations remain authoritative in runtime.field
```

Not every ENCODE output is a choice set. A sequence, dependency graph, or
evidence collection may contain many units without requiring suppression.

### 5.3 Proposed exact choice need

Working shape:

```lua
{
  kind = "choice_need",
  target_operator = "☳",
  causal_class = "blocking_demand",
  consumer_contract = string,
  choice_set_ref = string,
  alternative_ids = string[],
  alternative_versions = { [id] = integer },
  required_max_selected = integer,
  selection_policy_ref = string,
  scope_refs = exact_alternative_refs,
  provenance_refs = formation_and_consumer_refs,
}
```

`blocking_demand` is lawful only because the named consumer explicitly
requires fewer live alternatives than currently exist. A possibility set with
no cardinality requirement creates telemetry, not pressure.

### 5.4 CHOOSE action must not pre-execute CHOOSE

Pressure derivation must not secretly select the winner. That would collapse
router movement and semantic CHOOSE into one operation.

The route-carried action should define:

```text
the exact possibility set
the exact current versions
the maximum selected cardinality
the body-owned selection policy and its Packet-state reference
bounded killed-alternative sampling
```

☳ itself computes the selected and suppressed members under that frozen
contract. Arbitrary `semantic_ranking` supplied by the caller after route
commit is forbidden. A semantic proposal may influence choice only through a
stored, referenced Packet record whose truth status remains visible.

The proposed action mode is:

```text
alternative_collapse
```

### 5.5 Choice discharge

Discharge requires all of the following:

```text
one immutable choice event naming the pre-action set and versions
selected ids and every suppressed id
field activation mutations caused by that same event
post-action selected cardinality within the contract
killed-alternative count plus bounded detail sample
visible mandatory choice loss
effect refs matching the committed operand refs
```

The operand refs remain the pre-action exact versions. Post-action activation
increments the unit versions and is verified as the expected consequence, not
misreported as the original operand.

### 5.6 Confirmation

If the derived current set has one or zero live alternatives:

```text
choice_need is absent
☳ is not selected for collapse
no alternatives are killed
no CHOOSE identity loss is charged
```

A direct compatibility call may still return a typed confirmation payload, but
that payload is not promotion evidence for CHOOSE.

### 5.7 Choice re-arm

Choice may re-arm when:

```text
new alternatives enter the same declared set
membership changes under a new formation version
the consumer cardinality contract changes
the stored selection basis changes referent/version
two or more alternatives become live again through a lawful future operation
```

Unrelated global revision changes do not re-arm the need.

## 6. Pair Law Under Qualified Pressure

The revised pair is:

```text
☵ creates a receiver-suitable form
☵ explicitly declares whether that form contains a possibility set
☳ collapses only a declared set whose consumer requires collapse
```

Expected bounded route when all facts exist:

```text
☴ -> ☵ -> ☴ -> ☳ -> ☴
```

But each edge remains conditional:

```text
no receiver contract              -> no ☵ pressure
already formed at exact versions  -> no ☵ pressure
formation has no choice contract  -> no ☳ pressure
one current alternative           -> confirmation, no ☳ pressure
two alternatives allowed          -> no ☳ pressure
```

This is how the temporary eye rails become emergent observations rather than
hardcoded movement.

## 8. Falsifiers And False Greens

### Encoding falsifiers

```text
E0 source exists but no receiver contract: no encoding_need
E1 receiver contract exists but source is unobserved: OBSERVE wins first
E2 exact observed source lacks current formation: one encoding_need
E3 successful structure formation: same need disappears
E4 unrelated unit mutation: discharged need stays absent
E5 exact source version changes: need re-arms for that source only
E6 shadow/coarse identity map exists: production need remains
E7 caller replaces source scope or requested shape: loud invariant failure
E8 organ visits but writes no exact identity map/loss: not discharged
E9 relation-guided formation remains unchanged and non-shadow
```

### Choice falsifiers

```text
C0 two unrelated live units: no choice_need
C1 declared set with one live member: confirmation, no choice_need/loss
C2 declared set with two members but no collapse consumer: no choice_need
C3 consumer requires one of two current alternatives: one choice_need
C4 successful collapse: one selected, others suppressed, need disappears
C5 killed sample truncates: complete suppressed ids/count remain visible
C6 caller injects semantic ranking after route commit: loud invariant failure
C7 selected or suppressed operand version is stale: action rejected before effect
C8 choice event without matching activation effects: not discharged
C9 field order changes without referent change: no invented new need
C10 router selects an operator neighbor: no CHOOSE event or choice loss
```

### Pair falsifiers

```text
P0 every ENCODE output automatically becomes a choice set: reject
P1 every multi-unit field automatically selects ☳: reject
P2 compatibility language encoding qualifies promotion: reject
P3 semantic ranking from substrate silently becomes runtime truth: reject
P4 ☳ mutates identity/carrier instead of activation: reject
P5 exact ☵ or ☳ action changes live route in shadow mode: reject
```

## 9. Decisions Carried Forward

```text

D2 generic semantic-text ENCODE remains compatibility code, not promotion evidence
D3 production generic ENCODE consumes a bounded exact field scope
D4 encoding need requires a named receiver contract
D5 production formation records source versions, receiver form, identity map, and loss
D6 possibility-set identity is derived from immutable formation provenance
D7 CHOOSE requires at least two current alternatives and a collapse consumer
D8 route action identifies the set and policy but does not preselect the winner
D9 caller/substrate cannot replace action-owned scope or ranking after commit
D10 confirmation produces no qualified need and no identity loss
D11 exact object changes re-arm; unrelated global revision changes do not
D12 old count-based choice loss remains a visible provisional proxy
D13 default authority and full 38-direction corpus remain out of scope
```

## 10. Open Pressure For The Table

The table must make the following choices explicit enough for crystall:

```text
which first receiver contract qualifies generic structure formation
where its requested shape is declared in current Packet state
which immutable formation event carries the choice contract
which body-owned selection policy is admissible in v0
how pre-action operand refs and post-action activation versions share one effect proof
whether output disappearance re-arms ENCODE or creates separate repair pressure
```

The table must not solve these by adding numeric weights, LLM routing, a second
mutable possibility store, or a harness-owned scope.

## Working Claim

The next bounded treatment should not make ENCODE or CHOOSE more intelligent.

It should make their causality exact:

```text
named receiver needs exact form -> ☵
named consumer needs exact collapse -> ☳
```

The body owns the operands, the route carries the action, the organ owns the
effect, and only the matching effect discharges the need.
