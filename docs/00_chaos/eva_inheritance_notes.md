# Eva Inheritance Notes

Eva / planGOD is a laboratory, not the final body.

It contains useful organs, failures, and experimental traces. The body should
inherit mechanisms, not copy the old topology.

## Keep

```text
provider abstraction
explicit prompt assembly boundary
packet-like task flow
operator routing
parallel / chain / grok pressure modes
semantic anchors
glyph residues
runtime traces
no hidden steering rule
```

## Do Not Keep As-Is

```text
OBSERVE as one central brain
phantoms as prompt-only roles
RUNTIME as simple memory
history.json as memory
all lenses loaded as context
old microPL adjacency
prose-first internal state everywhere
```

## Lens Compiler Mechanism

The useful part is not old lenses themselves.
The useful part is the compilation mechanism:

```text
seed -> chain -> essence -> grok -> lens
```

Working reading:

```text
seed    = initial raw direction
chain   = cleaning / purification / noise burn
essence = cleaned result, not final lens
grok    = compression under semantic pressure
lens    = substrate-facing cognitive anchor
```

Core law:

```text
clean first
compress second
```

Compressing dirty material produces dense dirt.

## Memory Lesson

Eva showed that memory is not a pile of logs.

Better current reading:

```text
memory ~= fast decoding of prior residue
```

The body should preserve traces and residues, but runtime memory should be the
ability to reopen prior form under current constraints.

## Hallucination Lesson

Eva also showed that hallucination is not one thing.

Sometimes it is just false output.
Sometimes it is unsupported semantic drift.
Sometimes it exposes a missing route the body does not yet have.

The new body must distinguish these cases instead of treating all of them as
either truth or garbage.
